This documentation's itself

This webpage is an example of Docronym pages. It is fresh new framework, so there is not much people using it.

Its structure is just like this:

  1. index.html - The entry point of our webpage. It contains some settings and general structure of the page.
  2. Dposts - The folder that holds all the posts. Including
    • Customisation
    • Edit the whole page
    • Example pages
    • ...and all the posts you see on the left
  3. style files - I wanted it to look same in as much devices as possible. So I used a css preprocessor (SCSS). It brings some style files that brings a little complexity to the source code. Plus if you want to change style, you will need a SCSS compiler. But don't mind. Style and...
  4. Docronym.js files will be in index.html file. So all the code that you don't have to touch, will be hidden in index.html.

Don't forget, this is This page's structure, which is kinda source code. Yours will be like:

  1. index.html
  2. Dposts
    • Post name.html
    • Another post baby.html
